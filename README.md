# VPN Checker

Автоматизированный агрегатор, тестер и дистрибьютор прокси-серверов.
Проект поддерживает множество протоколов (VLESS, VMess, Trojan, Shadowsocks, Hysteria2) благодаря использованию ядра [sing-box](https://sing-box.sagernet.org/).

## Особенности
- **Мультипротокольность**: Нативная поддержка ссылок VLESS, VMess, Trojan, SS и Hysteria2.
- **Дедупликация**: Автоматическое удаление дубликатов по хосту и порту *перед* началом тестирования.
- **Автономная система тестирования**: Периодически (по расписанию) скачивает подписки, удаляет мертвые узлы и тестирует доступность через заданные тестовые URL (например, заблокированные сайты вроде `instagram`, `chatgpt`, `x`).
- **Стойкость к DPI**: Встроены жесткие проверки для обхода ТСПУ и геоблокировок РФ.
- **Zero-Downtime Webhook**: Бесшовное обновление списка серверов без перебоев для конечных пользователей даже во время активного сканирования. Раздача лучших отобранных конфигураций прямо в ваше VPN-приложение (сортировка по минимальному пингу и наибольшему числу successful tests, автоматическое переименование конфигураций в 1, 2, 3...).
- **Web-интерфейс**: Управление подписками, тестовыми URL, параметрами таймаутов, пинга, concurrency и сменой пароля через удобную админку (стек: FastAPI + Jinja2 + Tailwind CSS UI).
- **Умный парсинг**: Поддерживает не только прямые ссылки (raw.github), но и скачивающиеся по прямым ссылкам базы (base64 кодировки или plain text).

## Требования к серверу (VPS)

- **ОС**: Любой Linux дистрибутив (например, Ubuntu 22.04 / Debian 12 / Rocky Linux).
- **Docker и Docker Compose**: Установленные и настроенные.
- Минимально: 1 vCPU, 512MB RAM, 10GB SSD.

## Развертывание

1. **Клонируйте репозиторий**:
```bash
git clone https://github.com/GoGoButters/VLESS-Checker.git
cd VLESS-Checker
```

2. **Сборка и запуск контейнера**:
```bash
docker compose up -d --build
```

Система автоматически скачает ядро `sing-box`, установит зависимости Python, развернет базу данных `SQLite` (`data/vpn_checker.db` на хосте благодаря volume) и запустит web-сервер.

3. **Доступ к панели управления**:
Откройте браузер и перейдите по адресу:
```
http://<ВАШ_IP_АДРЕС>:8000
```
- **Логин по умолчанию**: `admin`
*(Обязательно поменяйте данные для входа в разделе **Settings** после первой авторизации!)*

## Настройка системы

После авторизации перейдите в **Settings**:
- Измените пароль администратора.
- Измените секретный путь для Webhook (по умолчанию: `secret-distrib`).
- Настройте лимит отдачи прокси.
- Укажите расписание автоматической проверки (в минутах).
- Укажите порог пинга и таймаута для ваших целей.

На вкладке **Test URLs** настроены (по умолчанию) сайты, через которые прокси будут проверяться:
- `https://www.instagram.com/favicon.ico`
- `https://x.com/favicon.ico`
- `https://chatgpt.com/favicon.ico`
- `https://rutracker.org/favicon.ico`
- `https://www.youtube.com/generate_204`

Прокси будет считаться валидным, если пройдет **минимум 1 тест**. Лучшими считаются те, кто прошел **наибольшее** количество тестов с наименьшим пингом.

На вкладке **Subscriptions** добавьте ваши источники прокси (URL ссылок).

## Обновление приложения

Если вышли обновления в репозитории:
```bash
cd VLESS-Checker
git pull
docker compose build --no-cache
docker compose up -d
```
База данных сохранится в volume `vpn_data` и не будет стерта при пересборке.

## Технологии
- FastAPI (Python 3.11+)
- SQLModel / SQLite (Асинхронные запросы не используются из-за ограничения sqlite, поток изолирован)
- Jinja2, Tailwind CSS (Dark Mode/Glassmorphism)
- [sing-box](https://github.com/SagerNet/sing-box)
