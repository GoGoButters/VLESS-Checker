# VPN Checker

Автоматизированный агрегатор, тестер и дистрибьютор прокси-серверов.
Проект поддерживает множество протоколов (VLESS, VMess, Trojan, Shadowsocks, Hysteria2) благодаря использованию ядра [sing-box](https://sing-box.sagernet.org/).

## Особенности
- **Мультипротокольность**: Нативная поддержка ссылок VLESS, VMess, Trojan, SS и Hysteria2.
- **Deduplication**: Автоматическое удаление дубликатов по хосту и порту *перед* началом тестирования.
- **Автономная система тестирования**: Периодически (по расписанию) скачивает подписки, удаляет мертвые узлы и тестирует доступность через заданные тестовые URL (например, `gstatic`, `google`, `youtube`).
- **Web-интерфейс**: Управление подписками, тестовыми URL, параметрами таймаутов, пинга и concurrency через удобную админку.
- **Умный парсинг**: Поддерживает не только прямые ссылки (raw.github), но и скачивающиеся по прямым ссылкам базы (base64 кодировки или plain text).
- **Публичный Webhook**: Раздача лучших отобранных конфигураций прямо в ваше VPN-приложение (сортировка по минимальному пингу и наибольшему числу successful tests, автоматическое переименование конфигураций в 1, 2, 3...).

## Требования к серверу (VPS)

- **ОС**: Любой Linux дистрибутив (например, Ubuntu 22.04 / Debian 12 / Rocky Linux).
- **Docker и Docker Compose**: Установленные и настроенные.
- Минимально: 1 vCPU, 512MB RAM, 10GB SSD.

## Развертывание

1. **Установите Docker** (если еще не установлен):
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo systemctl enable --now docker
sudo apt-get install docker-compose-plugin
```

2. **Клонируйте репозиторий**:
```bash
git clone https://github.com/GoGoButters/VLESS-Checker.git
cd VLESS-Checker
```

3. **Сборка и запуск контейнера**:
```bash
docker compose up -d --build
```

Система автоматически скачает ядро `sing-box`, установит зависимости Python, развернет базу данных `SQLite` (`data/db.sqlite3` на хосте благодаря volume) и запустит web-сервер.

4. **Доступ к панели управления**:
Откройте браузер и перейдите по адресу:
```
http://<ВАШ_IP_АДРЕС>:8000
```
- **Login**: `admin`
- **Password**: `admin`

*(Обязательно поменяйте данные для входа в `config.yml` или через переменные окружения, если они добавлены! В текущей реализации пароли захардкожены в `main.py` или `config.py` — убедитесь, что вы изменили их перед публикацией).*

## Настройка системы

После авторизации перейдите в **Settings**:
- Измените секретный путь для Webhook (по умолчанию: `my-secret-webhook`).
- Настройте лимит отдачи прокси.
- Укажите расписание автоматической проверки (в часах/днях).
- Укажите порог пинга и таймаута для ваших целей.

На вкладке **Test URLs** настройте сайты, через которые прокси будут проверяться:
- `http://www.gstatic.com/generate_204`
- `https://google.com`
- И так далее. Прокси будет считаться валидным, если пройдет **минимум 1 тест**. Лучшими считаются те, кто прошел **наибольшее** количество тестов с наименьшим пингом.

На вкладке **Subscriptions** добавьте ваши источники прокси (URL ссылок).

## Обновление приложения

Если вышли обновления в репозитории:
```bash
cd VLESS-Checker
git pull
docker compose up -d --build
```
База данных сохранится в volume `vpn_data` и не будет стерта при пересборке.
